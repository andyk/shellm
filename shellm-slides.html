Architecture Deep Dive

A bash-native AI agent that thinks by writing shell commands, running them, and iterating until it has an answer. Composed of dead-simple shell tools.

pure bash ~5,000 lines append-only logs text is the interface

Key Ideas Design Philosophy

Ken Thompson’s Unix Shell Philosophy

Each tool does one thing well. Favor simplicity, clarity, and minimal, composable programs. shellm + llm + traj + context + mem + skills — each a standalone CLI, each composable via stdin/stdout/files.

Bash CLI implementation of RLM

A Recursive Language Model agent implemented in bash. The LLM generates code, observes its output, and recurses. Collapses all tool calling into one tool: the shell itself. No need to explicitly declare tools — the entire Unix environment is the toolbox.

Docker by default, lightweight

Sandboxed execution in Docker containers. Container re-use across runs keeps it fast — no cold-start penalty. Easy to plug in other sandbox solutions (local, DinD, broker).

Trajectory DAG abstraction

Agent inference context is a function of its trajectory. Append-only JSONL logs form a DAG with fork/merge. Non-destructive compaction: select head + tail steps, elide the middle. Full history preserved.

Python venv-like semantics

Activate an agent similar to how you activate a Python env. Trivial for humans and agents to co-work in the same shell. Switch identities like switching Python virtual environments.

Persistent Agency

Supports both traditional human/agent turn-taking and Persistent Agency where the agent thinks continuously and autonomously like a human does; human messages are observations injected into the stream asynchronously.

Why the Shell? Philosophy

LLMs are text-in, text-out. The Unix shell is where everything is text — stdin, stdout, pipes, files, env vars.


Most agent frameworks give the LLM a curated menu of function calls. If your menu doesn’t include a capability, the model can’t do it.


shellm inverts this. Instead of enumerating tools, drop the LLM into a composable environment and let it figure out what to do.

The Unix advantage

  • curl is the HTTP client
  • jq is the JSON processor
  • python3 -c is the escape hatch
  • No schemas, no wrappers
  • Compose tools by piping them together

Key insight

The model composes tools the same way a human would.

01

shellm Agent Loop

  • Build messages from trajectory via context
  • Call LLM with system prompt + messages via llm
  • Extract ```bash code block from response
  • Execute code in isolated workspace (Docker or local)
  • Capture stdout, stderr, exit code
  • Append reasoning + shell-output steps to trajectory
  • Repeat until FINAL is set or max iterations reached

What you see

▶ Iteration 1 — calling gpt-5.5...
▶ Executing bash (12 lines):
    curl -s "https://api.example.com"
    | jq '.results'
  │ {"count": 42, "items": [...]}
  $ FINAL="Found 42 results"
  Exit 0

▶ Final answer received
Found 42 results

Safety

  • Inactivity watchdog (30s default)
  • Docker sandboxing with broker mode
  • Package install interception

03

Trajectories traj

Append-only JSONL step logs. Every action, thought, and observation is recorded.

Step format

{
  "type": "reasoning",
  "step_id": "abc12345-...",
  "ts": "2026-06-16T14:30:22-0700",
  "thought": "I should check the API",
  "cmd": "curl https://..."
}

Step types

trajectory prompt reasoning shell-output final thought observation fork merge

Directory structure

trajectories/ ├── 42a2c63d-root/ │ ├── trajectory.jsonl # main log │ ├── blobs/ # large output │ │ └── <step_id>.stdout │ └── 920b79eb-subtask/ # child │ └── trajectory.jsonl └── be610f6e-explain-quicksort/ └── trajectory.jsonl

Blob spilling

stdout/stderr > 4KB → written to blobs/, inline truncated, *_ref field added.

traj CLI Commands

CommandPurpose

Skills Capabilities

SKILL.md-based agent abilities. Markdown with YAML frontmatter. No code generation framework needed.

SKILL.md format

---
name: web-fetch
description: Fetch web content
metadata:
  shellm:
    requires:
      bins: ["curl", "jq"]
---

# web-fetch

Fetch and parse web content from URLs.

## Commands
  fetch URL          Fetch raw content
  fetch --json URL   Parse as JSON

## Examples
```bash
#| eval: true
curl https://example.com | head -5
```

Commands

skills install ./my-skill
skills install https://github.com/...
skills list
skills show web-fetch
skills promote web-fetch  # to kernel
skills search "web"

How skills work

  • skills prompt outputs all skill descriptions
  • Included in system prompt for LLM calls
  • LLM reads skill docs and uses commands
  • Kernel skills (mem, chat) always available
  • Local skills installed per-identity
  • Executable bash blocks with #| eval: true

Thinker Structure Details

Directory layout

thinkers/executive/ ├── step # receives step JSON on stdin ├── subscriptions.jsonl # type filters ├── prompt.md # system prompt template ├── start # optional: long-running setup └── stop # optional: cleanup

Subscriptions

# subscriptions.jsonl
{"types":["thought","action","observation",
  "human-msg","agent-msg","merge"],
 "trigger_self": false}

Self-triggering prevention

Every step must include "source":"<thinker-name>". Dispatcher skips delivering a step to the thinker that wrote it (unless trigger_self: true).

Key Architectural Patterns Design

Text is the interface

All state lives in files: JSONL, markdown, bash scripts. No databases. Everything is text that humans can read and LLMs can process.

Environment-driven discovery

Tools discover context via env vars set by identity activate. No config files to parse, no service discovery.

Append-only logs

Trajectories are immutable. Steps are never modified after creation. History is always complete and verifiable.

Reactive composition

Thinkers subscribe to step types. The trajectory is the event bus. Add/remove thinkers independently without changing the core.

Isolation by default

Each identity is a separate namespace. Docker provides process isolation. Workdirs provide filesystem isolation. Nothing bleeds.

Unix composability

Each tool does one thing well. shellm + traj + context + mem + skills = full agent stack. Compose via stdin/stdout/files.

Getting Started

Install

git clone https://github.com/andyk/shellm.git
cd shellm
./install.sh --symlinks

Quick start

export ANTHROPIC_API_KEY="sk-ant-..."

# Run a task
shellm what is the mass of jupiter

# Pipe data in
cat data.csv | shellm summarize this

# Create an identity
identity new myagent
identity shell myagent

# Start thinking
thinkers start
chat send "hello, what are you thinking?"
